# TRON 归集系统客户版安全说明书（1页）

## 1. 系统定位
本系统为**本地私有化部署**的钱包归集管理系统，支持 TRX 与 USDT(TRC20) 的地址管理、资产同步与归集任务执行。  
系统默认部署于客户自有服务器/终端，不上传私钥到第三方托管平台。

## 2. 安全设计原则
1. 最小暴露面：仅开放必要端口，建议内网访问。
2. 密钥保护优先：私钥不明文保存，落库前加密。
3. 可追溯：关键操作保留审计日志。
4. 可恢复：支持数据库与配置备份恢复。

## 3. 关键安全能力
1. 私钥加密存储：私钥在写入数据库前进行对称加密，数据库泄露不等于私钥明文泄露。
2. 本地化部署：数据、密钥、日志由客户侧控制，不经过外部托管。
3. 操作审计：地址导入/导出、批量删除、归集任务等关键动作可记录。
4. 网络可控：可配置为仅内网访问、反向代理、白名单与 HTTPS。
5. 运行可管控：支持按客户规范进行备份、轮换、权限控制与安全巡检。

## 4. 责任边界（必须明确）
1. 乙方（交付方）负责：系统安全能力实现、部署指导、配置建议与风险告知。
2. 甲方（客户）负责：主机安全、运维账号、网络边界、防病毒、补丁更新、备份执行、密钥保管制度。

## 5. 风险声明
1. 不承诺“零风险”或“绝对安全”。
2. 若客户侧存在弱口令、主机中毒、远程桌面泄露、私钥文件外传等情况，均可能导致资产风险。
3. 建议客户建立双人复核与分级审批流程，降低误操作与内部风险。

## 6. 上线最低安全基线（建议）
1. 禁止免登录模式在生产环境使用。
2. 管理后台仅内网可访问，启用 HTTPS。
3. 强口令+定期轮换，限制管理员人数。
4. `.env` 与数据库分离备份，主密钥独立保管。
5. 导出私钥文件仅离线加密保存，禁止通过聊天工具传输。
6. 至少每日一次数据库备份，至少每月一次恢复演练。
